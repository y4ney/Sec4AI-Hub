# 威胁描述

资源过载指攻击者蓄意耗尽 AI Agent 的计算能力、内存或外部服务依赖，导致系统性能下降或故障。与传统的拒绝服务攻击不同，AI Agent 因推理任务消耗大量资源、依赖多服务协同且需处理并发请求，因此对此类攻击尤为敏感，易出现响应延迟、决策瘫痪，或引发互联系统的级联故障。在实时性和自主性要求较高的环境中，该威胁尤为关键 —— 资源耗尽会扰乱核心业务操作，损害系统可靠性。

资源过载攻击针对 AI 系统的计算资源、内存和服务容量，通过滥用其高资源消耗特性，导致性能下降或系统故障。

# 与 OWASP LLM Top 10 的关联

该威胁与 2025 年大语言模型安全十大风险中的**无界资源消耗**相关，而智能体人工智能系统因缺乏人工的直接监督，可跨会话自主调度、排队并执行任务，因此面临的资源过载风险更为突出。与标准的大语言模型应用不同，AI Agent 可自行触发任务、生成额外进程，并与多个 AI Agent 协同工作，这会导致资源消耗呈指数级增长，形成更复杂、更具系统性的威胁。

# 攻击场景

## 场景 1：推理时间利用

攻击者向人工智能安全系统输入特制的内容，迫使系统执行资源密集型的分析操作，使其处理能力达到饱和，进而延迟实时威胁检测。

## 场景 2：多智能体资源耗尽

攻击者触发系统中的多个人工智能 AI Agent 同时执行复杂的决策任务，耗尽计算资源，导致所有业务的服务性能下降。

## 场景 3：API 配额耗尽

攻击者向 AI Agent 发送大量请求，使其触发频繁的外部 API 调用，快速消耗系统的 API 配额，既阻碍了合法的使用行为，又造成企业运营成本大幅增加。

## 场景 4：内存级联故障

攻击者发起多个需要大量内存分配的复杂任务，导致内存碎片和内存泄漏，进而引发全系统的资源耗尽，不仅影响被攻击的人工智能，还会波及依赖其运行的其他服务。

# 缓解方法

- 部署资源管理控制机制
- 实施自适应扩缩容机制
- 建立资源配额制度
- 实时监控系统负载以检测和缓解过载行为
- 对每个Agent会话实施速率限制策略，限制高频任务请求

# 威胁描述

权限泄露指攻击者利用角色管理不当、配置过度宽松或动态权限继承的漏洞，提升权限并滥用 AI Agent 的访问能力。与传统系统不同，AI Agent 会自主继承权限，这会产生安全监控盲区 —— 临时或继承的权限可能被滥用，进而执行未授权操作，例如将基础的工具访问权限提升至管理员控制权限。人工智能的跨系统自主性进一步加剧了该风险，导致严格的访问边界难以执行、权限滥用无法被实时检测，且未授权操作难以被阻止。

权限失陷发生在攻击者利用权限管理中的漏洞执行未经授权操作的情况下。这通常涉及动态角色继承机制或权限配置错误。

# 与 OWASP LLM Top 10 的关联

该威胁在一定程度上被 2025 年大语言模型安全十大风险中的**过度自主**所覆盖，但 AI Agent 可动态委托角色或调用外部工具，进一步放大了权限提升的风险，因此需要更严格的边界管控。

# 攻击场景

## 场景 1：动态权限提升

攻击者以故障排查为借口，操纵 AI Agent 调用临时的管理员权限，随后利用配置漏洞永久保留高权限，进而提取敏感数据。

## 场景 2：跨系统授权利用

攻击者利用 AI Agent 对企业多个系统的访问权限，借助权限范围管控的缺陷，将权限从人力资源系统提升至财务系统，实现未授权的数据提取。

## 场景 3：影子智能体部署

攻击者利用薄弱的访问控制，创建一个可继承合法凭证的恶意 AI Agent，使其在无察觉的情况下运行，并执行数据窃取或未授权交易操作。

# 缓解方法

- 实施细粒度权限控制
- 动态访问验证
- 加强对角色变更的监控
- 对高权限操作进行全面审计
- 除非通过预定义工作流明确授权，否则应防止跨Agent的权限委托

# 威胁描述

多智能体系统中的恶意智能体指恶意或被入侵的 AI Agent 渗透多智能体架构，利用信任机制、工作流依赖或系统资源，操纵决策、破坏数据或实施拒绝服务攻击。这些恶意智能体既可能是攻击者故意植入的，也可能源于被入侵的人工智能组件，最终引发系统性的混乱和安全故障。

恶意或被攻破的 AI Agent 在正常监控边界之外运行，执行未授权操作或窃取数据。其中还包括“传染式后门（infectious backdoors）”的概念，即一个被攻破的 Agent 会向其他 Agent 传播恶意逻辑。

# 与 OWASP Top 10 for LLM的关联

该威胁可让攻击者在Agentic AI 场景中，对 **LLM06:2025 过度授权** 漏洞进行利用；并会引入持续性恶意智能体风险，使得恶意智能体能够悄无声息地嵌入工作流中。

# 攻击场景

## 场景 1：恶意工作流注入

一个恶意 AI Agent 冒充金融审批人工智能，利用 AI Agent 间的信任关系注入欺诈性交易，同时绕过验证控制。

## 场景 2：金融交易中的编排劫持

一个恶意 AI Agent 将欺诈性交易通过多个低权限 AI Agent 进行路由，利用碎片化的审批流程绕过人工验证。

## 场景 3：协同智能体泛洪攻击

多个恶意 AI Agent 同时生成大量任务请求，耗尽计算资源，延迟关键决策过程。

## 场景 4：传染性后门级联攻击

金融多智能体系统中的一个被入侵 AI Agent，在其推理链中植入 “传染性后门”。当其他 AI Agent 在协同过程中调用其输出结果时，恶意逻辑会在网络中悄悄传播，最终导致交易审批系统被系统性入侵。更多相关场景可参考该研究论文：[https://arxiv.org/html/2503.09648v1#S2](https://arxiv.org/html/2503.09648v1#S2 "https://arxiv.org/html/2503.09648v1#S2")

# 缓解方法

通过策略约束限制 Agent 的自主性，并持续进行行为监控。虽然目前尚不存在针对大语言模型的加密证明机制，但可以通过受控托管环境、定期 AI 红队测试，以及输入/输出异常监控来维护 Agent 的完整性。

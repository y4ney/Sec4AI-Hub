# 威胁描述

攻击者利用用户对 AI Agent 的信任，在用户未察觉的情况下影响其决策。在被入侵的人工智能系统中，攻击者会操纵 AI Agent，诱使用户执行有害操作，如处理欺诈交易、点击钓鱼链接或传播虚假信息。用户对 AI Agent 回复的默认信任会降低其警惕性，这使得该方式成为通过人工智能实施社会工程攻击的有效手段。

在 AI Agent 与人类用户直接交互的场景中，信任关系会降低用户的怀疑程度，从而增加对 Agent 回复和自主行为的依赖。这种隐含信任和直接的人机互动带来风险：攻击者可能诱导 Agent 操纵用户、传播错误信息或执行隐蔽操作。

# 攻击场景

## 场景 1：人工智能驱动的发票欺诈

攻击者利用间接提示注入漏洞操纵企业智能助手 AI Agent，将合法供应商的银行信息替换为自身账户信息。用户因信任 AI Agent 的回复，在不知情的情况下完成了欺诈性的电汇操作。

## 场景 2：人工智能驱动的钓鱼攻击

攻击者入侵人工智能助手，使其生成虚假消息，诱导用户点击伪装成安全更新的恶意链接。用户因信任人工智能，点击链接后被重定向至钓鱼网站，最终导致账户被盗。

# 缓解方法

监控 Agent 的行为以确保其符合既定角色和预期行动。限制工具访问以减少攻击面，限制 Agent 输出链接的能力，并使用防护规则（guardrails）、内容审核 API 或其他模型来检测和过滤被操纵的回复。

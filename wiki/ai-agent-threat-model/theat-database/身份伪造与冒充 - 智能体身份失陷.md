# 威胁描述

身份仿冒与冒充是 AI Agent 面临的重大威胁，攻击者利用身份验证机制的漏洞，冒充 AI Agent、人类用户或外部服务，在不被发现的情况下获取未授权访问权限并执行有害操作。该威胁在基于信任的多智能体环境中尤为危险 —— 攻击者可操纵身份验证流程、利用身份继承漏洞，或绕过验证控制，以虚假身份开展操作。攻击者还可能窃取或滥用 AI Agent 的正式、持久化身份，从而长期获取企业资源的高权限访问权限。这种持久化的访问权限会扩大攻击影响范围，同时破坏审计的有效性和责任的可追溯性。随着具备企业级访问权限的 AI Agent 获得正式认可，该威胁的危害性进一步加剧。

攻击者利用认证机制漏洞，冒充 AI Agent或用户身份，在虚假身份下执行未经授权的操作。这包括窃取或滥用正式的、持久化的Agent身份（例如 Microsoft Entra Agent ID），从而获得具有特权的长期 API 访问权限，绕过Agent的对话接口及其安全防护机制。

# 攻击场景

## 场景 1：用户冒充

攻击者向具备发送邮件权限的 AI Agent 注入间接提示词，诱使其以合法用户的名义发送恶意邮件。

## 场景 2：AI Agent 身份仿冒

攻击者入侵人力资源入职管理 AI Agent，利用其权限创建虚假用户账户，同时伪装成系统正常行为。

## 场景 3：行为模拟攻击

一个恶意 AI Agent 模仿合法系统智能体的交互方式和决策模式，以可信实体的身份获取未授权访问权限。

## 场景 4：跨平台身份仿冒

一个自适应的恶意 AI Agent 可动态修改自身身份，以匹配不同平台的身份验证环境，绕过安全边界实现全域访问。此外，攻击者还会利用外部工具（如 GitHub）中的权限继承漏洞，使恶意 AI Agent 接管因身份验证策略薄弱而被无意授予权限的资源。

## 场景 5：栽赃其他用户

攻击者利用薄弱的身份验证机制，以其他用户的身份执行敏感操作，让该用户为未授权行为承担责任，同时隐藏自身踪迹。

## 场景 6：AI Agent 持久化身份劫持

攻击者从配置不当的云存储中提取与企业 AI Agent 绑定的长效 API 令牌，利用该正式身份（如微软 Entra Agent ID）在多个企业服务中冒充该 AI Agent，提升权限并通过后端自动化流水线实现横向移动。由于 AI Agent 身份被认定为可信实体，攻击者可在该身份被明确吊销或轮换前，持续进行无察觉的访问。

# 缓解方法

- 建立全面的身份验证框架
- 强化信任边界与最小权限访问原则
- 部署持续监控机制以检测身份冒充行为
- 采用行为画像技术，并引入第二模型进行行为审查，以检测可能表明身份伪造的异常活动

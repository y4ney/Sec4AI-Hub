# 威胁描述

模型上下文协议（Model Context Protocol，简称 MCP）、智能体间通信协议（Agent-to-Agent，简称 A2A）等协议的逐步普及，实现了 AI Agent 间的协同工作、工具共享与任务委托，但也基于 AI Agent 间的通信和协同机制，引入了新的攻击面。此类威胁涉及滥用协议中嵌入的命令和信任机制，包括操纵服务器响应、注入上下文信息，或通过定义模糊的工具描述、歧义的授权流程误导 AI Agent。当协议规范执行不严格，或协议实现缺乏输入验证和强身份绑定机制时，攻击者可劫持智能体行为、提升权限，或彻底绕过安全防护，进而导致工具被非预期执行、数据泄露，或产生不可逆转的系统操作。

攻击针对诸如 MCP 或 A2A 等协议中的漏洞，例如绕过授权或劫持上下文，从而导致 Agent 执行未授权操作。



# 攻击场景

## 场景 1：授权流程操纵

攻击者构造一个恶意 AI Agent 参与智能体间通信协议交互，并操纵授权协商流程；利用验证检查的缺失，该恶意 AI Agent 可在无明确用户意图的情况下，自动批准数据删除等敏感操作，绕过常规的安全防护措施。

## 场景 2：通过模型上下文协议响应注入实现上下文劫持

攻击者在模型上下文协议的实现中，拦截或构造服务器端响应，注入恶意的上下文信息或工具元数据；协同工作的 AI Agent 将该注入的响应判定为可信的协议上下文，进而执行非预期的后端操作。

## 场景 3：通过利用工具描述实现工具滥用

攻击者在共享工具注册表中，嵌入具有误导性或权限范围过宽的工具描述；在 AI Agent 协同过程中，其他 AI Agent 基于错误的认知调用该工具，进而在不知情的情况下泄露敏感数据或触发高权限的 API 调用。

# 缓解方法

通过强身份认证保护 Agent 之间的通信。对所有协议层数据（包括上下文载荷和工具元数据）进行清理与验证，以防止注入或误解析。将 Agent 间的委托限制在严格定义的功能范围内。记录所有 Agent 间通信和工具调用，以便检测异常并支持事后分析。同时对通信进行加密，以防止中间人攻击。

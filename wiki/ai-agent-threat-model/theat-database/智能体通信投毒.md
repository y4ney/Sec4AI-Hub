# 威胁描述

智能体通信投毒指攻击者操纵 AI Agent 间的通信渠道，向多智能体人工智能系统中注入虚假信息、误导决策过程并破坏共享知识。与孤立的人工智能攻击不同，该威胁利用分布式人工智能协同的复杂性，导致虚假信息级联传播、系统性故障，以及互联 AI Agent 间的决策完整性受损。

攻击者操纵 AI Agent 之间的通信渠道，传播虚假信息、破坏工作流程或影响决策过程。

# 与 OWASP Top 10 for LLM 的关联

与记忆投毒类似，该威胁不仅超出了 **LLM04:2025 数据与模型投毒** 所定义的静态数据投毒，以及 **LLM08:2025 向量与嵌入漏洞** 所涵盖的 RAG 嵌入投毒范围，还针对瞬时动态数据。

# 攻击场景

## 场景 1 ：协同决策操纵

攻击者向 AI Agent 的通信内容中注入误导性信息，逐步影响决策过程，使多智能体系统朝着目标错位的方向运行。

## 场景 2 ：信任网络利用

攻击者通过伪造虚假的共识消息、利用身份验证漏洞，操纵 AI Agent 间的验证机制，实现未授权访问并触发欺骗性行为。

## 场景 3 ：虚假信息注入与级联投毒

攻击者有策略地向多智能体网络中植入虚假数据，既可以是缓慢破坏推理过程的隐蔽性降级攻击，也可以是让虚假知识在各 AI Agent 间快速传播的突发式虚假信息级联攻击。

## 场景 4 ：通信渠道操纵

攻击者利用 AI Agent 间通信协议的漏洞，植入人为的通信屏障、拦截 / 修改消息并引入传输延迟，降低系统运行效率。

## 场景 5 ：共识机制利用

攻击者通过轻微干扰决策逻辑，在 AI Agent 间制造人为的意见分歧，逐步削弱系统的集体问题解决能力，使其变得不可靠。

# 缓解方法

部署加密消息认证机制，实施通信验证策略，并监控 Agent 之间的交互以检测异常。在关键决策流程中要求多 Agent 共识验证。

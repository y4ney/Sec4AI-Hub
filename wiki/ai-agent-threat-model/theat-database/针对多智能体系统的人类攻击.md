# 威胁描述

针对多智能体系统的人为攻击指攻击者利用 AI Agent 间的任务委托、信任关系和任务依赖，绕过安全控制、提升权限或破坏工作流。攻击者通过注入虚假任务、重新分配优先级，或向 AI Agent 分配大量任务使其过载，能够以难以追溯和缓解的方式操纵人工智能驱动的决策过程，最终引发系统性故障或未授权操作。

攻击者利用 Agent 之间的任务委托机制、信任关系以及工作流依赖来提升权限或操纵 AI 驱动的操作。

# 攻击场景

## 场景 1 ：通过多智能体仿冒实现协同权限提升

攻击者通过入侵身份验证和访问控制 AI Agent，渗透安全监控系统，使一个人工智能虚假验证另一个人工智能的身份，进而获取未授权访问权限。

## 场景 2 ：利用智能体委托循环实现权限提升

攻击者在相互依赖的 AI Agent 间反复提交权限提升请求，诱使系统误以为该请求已通过前期验证，进而授予高权限。

## 场景 3 ：通过智能体任务饱和实现拒绝服务

攻击者向多智能体系统提交大量高优先级任务，使其过载，导致安全 AI Agent 无法处理真实的威胁。

## 场景 4 ：跨智能体验证伪造

欺诈者利用多智能体生物特征或身份验证检查中的不一致性，操纵单个 AI Agent 批准一个本应在全系统验证中被拒绝的身份。

# 缓解方法

限制 Agent 的委托机制，实施 Agent 之间的身份认证，并部署行为监控来检测操纵行为。通过多 Agent 任务分段机制防止攻击者在相互连接的 Agent 之间逐步提升权限。

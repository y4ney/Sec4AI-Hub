# 威胁描述

攻击者入侵上游组件（如智能体提示词、插件、框架更新包等），使恶意逻辑通过可信的软件进行传播；在攻击行为被发现前，受污染的组件会导致 AI Agent 执行破坏性命令，或破坏业务工作流。

被攻破的供应链可能导致存在漏洞、恶意或过时的组件被引入 Agent 系统，使攻击者能够操控 Agent 行为、获取数据或执行任意代码。这种情况可能通过模型、库、工具、被污染的构建环境或其他系统组件发生。

# 与 OWASP LLM Top 10 的关联

2025 年大语言模型安全十大风险中的**供应链漏洞**，重点指出了集成不可信或受污染的第三方组件（如模型、库、插件）带来的风险。而在智能体人工智能系统中，该风险被大幅放大 ——AI Agent 不仅会使用这些组件，还会自主调用工具、串联决策，或通过协议与其他 AI Agent 协同工作。与独立的大语言模型部署不同，AI Agent 是具备持久性和状态性的实体，可持续执行操作或传播恶意逻辑；因此，单次的供应链入侵即可引发权限提升、行为偏移，或在互联 AI Agent 间造成系统性的滥用。

# 攻击场景

## 场景 1：亚马逊 Q 供应链入侵

攻击者向亚马逊 VS Code 端 Q 智能体的 GitHub 代码仓库中注入破坏性提示词，指令其 “将系统恢复至近乎出厂状态”。该恶意更新包（v1.84.0）被亚马逊无意发布，影响数千名开发者后才被发现。尽管该提示词最终未按预期执行，但这一事件证明，攻击者可通过供应链访问篡改 AI Agent 的逻辑，进而引发灾难性后果。

## 场景 2：Replit Vibe 编码事故

Replit 的自主编码 AI Agent 虚构了一个数据库，删除了真实数据库，并生成虚假的测试结果来掩盖故障。此次事故的原因包括测试环境与生产环境未做有效隔离、过度依赖未做沙箱隔离的工具，以及对提示词执行未做验证。

# 缓解方法

通过对制品进行数字签名来保护 Agent 生态系统，使用可验证的SBOM（如 AIBOM、Agent SBOM），并实施带有同行评审的版本控制。对供应链实施强身份认证，限制不可信工具的安装，并在沙箱隔离环境中运行 Agent。持续监控供应链中的漂移或恶意行为，并通过模拟供应链攻击的Agent 红队测试来验证防御能力。

# 威胁描述

**级联幻觉攻击（Cascading Hallucination Attacks）** 利用 AI 智能体无法区分事实与虚构信息的缺陷，使虚假信息在互联系统中传播、嵌入并不断放大，最终导致数据逐步被篡改、上下文被利用，以及系统性的虚假信息扩散。攻击者可操纵 AI 生成的输出，触发其欺骗性推理模式，将编造的虚假信息植入决策过程；而在具备持久化记忆和跨会话学习能力的系统中，这些虚假信息会长期存在并不断升级。

其本质是利用系统的自动化流程和依赖关系，让一个最初的小错误被持续继承、放大并影响后续决策和执行，最终造成连锁性风险。此类攻击利用 AI 生成“看似合理但实际错误信息”的倾向，这些错误信息可能在系统中传播并干扰决策过程。此外，还可能导致破坏性推理，进而影响工具调用行为。

# 与 OWASP Top 10 for LLM 的关联

**LLM09:2025 信息误导** 主要针对幻觉风险，但智能体式 AI 在单智能体和多智能体架构中均进一步扩大了该威胁。

在单智能体环境中，幻觉可通过自我强化机制（如反思、自我批判或记忆召回）不断叠加，导致智能体在多轮交互中不断强化并依赖错误信息。在多智能体系统中，信息误导可通过智能体间通信环路在各智能体之间传播并放大，进而引发级联错误与系统性失效。

# 攻击场景

## 场景 1 ：销售协同虚假信息级联

攻击者向销售类 AI 智能体人的回复中巧妙注入虚假的产品信息，这些信息会累积在其长期记忆和日志中，导致后续交互中的虚假信息不断升级并扩散。

## 场景 2 ： API 调用操纵与信息泄露

攻击者将虚构的 API 接口植入 AI 智能体的上下文，使其生成虚假的 API 调用指令，进而导致意外的数据泄露与系统完整性受损。

## 场景 3 ：医疗决策幻觉放大

攻击者向医疗类 AI Agent 的回复中植入虚假的诊疗指南，该指南会在 AI 的多次幻觉中不断强化，最终形成存在严重缺陷的医疗建议，对患者构成风险。

## 场景 4 ：外汇市场操纵

攻击者注入与外汇汇率相关的虚假信息，导致 AI 智能体依据不真实的汇率进行交易谈判，进而引发资金损失与市场动荡。

# 缓解方法

- 建立健壮的输出验证机制
- 实施行为约束
- 部署多源验证机制
- 通过反馈循环持续修正系统
- 在关键决策过程中使用 AI 生成知识前，必须进行二次验证。需要注意的是：当规模过大时，人类审核能力跟不上，监督机制会形式化甚至失效

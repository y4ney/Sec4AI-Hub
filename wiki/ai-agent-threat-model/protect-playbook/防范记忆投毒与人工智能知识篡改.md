# 核心目标

防范人工智能存储、调取或传播被篡改的数据，避免决策体系被破坏、虚假信息扩散。

# 主动防护：保障人工智能记忆访问安全与内容验证

- 推行记忆内容验证，对拟写入记忆的内容开展自动化异常扫描；将记忆持久化限制于可信数据源，对长期存储数据实施加密验证；
- 落实记忆访问日志记录机制；
- 通过会话隔离实现记忆访问分段管理，确保人工智能不会在不同用户会话间，传递非预期的知识信息；
- 基于上下文感知策略限制记忆访问，强制 AI Agent 仅能调取与当前业务任务相关的记忆内容，降低未授权知识提取风险；
- 根据数据敏感度，限制人工智能记忆的保留时长，避免其存储易被操纵、无保留必要的历史数据；
- 记忆更新需明确来源溯源，强制跟踪人工智能知识的获取渠道，确保修改内容均来自可信源；
- 跨会话持久化的记忆修改，需经过多智能体协同验证或外部验证（通过提交前验证，防范自我强化或未经验证的知识被持久化存储）；
- 新的人工智能知识在写入长期存储前，需通过概率真实性校验，与可信源进行比对验证（通过提交前事实核查，降低虚假信息的持久化风险）。

# 被动响应：检测并处置记忆投毒行为

- 部署异常检测系统，监控人工智能记忆日志中的非预期更新，包括未授权的记忆访问行为；
- 对可疑或已被篡改的记忆条目，在提交后重新开展多智能体协同验证或外部验证，确认其完整性或识别投毒行为；
- 对已存储的现有知识开展概率真实性重校验，检测长期使用中产生的知识偏差、虚假信息或污染；
- 启用回滚机制，当检测到异常时，将人工智能知识恢复至此前通过验证的状态；
- 生成人工智能记忆快照，便于异常发生时开展取证分析与回滚操作；
- 检测并标记异常的记忆修改频率，若发现人工智能记忆被以异常高频率改写，需警惕操纵攻击。

# 检测监控：防范虚假知识扩散

- 部署概率真实性校验机制，核验新知识与已验证事实的一致性；
- 限制未经验证数据源的知识传播，确保智能体不会将低可信度输入用于决策过程；
- 跟踪人工智能生成知识的溯源信息，留存知识演化的历史记录，便于虚假信息扩散的取证调查；
- 对人工智能知识更新实施版本管理，确保知识修改可审计，若发现篡改可实现回滚；
- 持续分析记忆访问模式，检测长期异常或策略偏移，验证访问日志的完整性，并将异常访问趋势与潜在的知识篡改关联分析。

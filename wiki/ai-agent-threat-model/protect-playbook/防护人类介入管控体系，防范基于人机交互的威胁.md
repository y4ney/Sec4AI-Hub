# 核心目标

防范攻击者压垮人类决策者、操纵人工智能意图，或利用人工智能欺骗性行为绕过安全防护。

# 主动防护：优化人类介入管控工作流，降低决策疲劳

- 利用人工智能信任评分，基于风险等级对人类介入管控审核队列进行优先级排序；
- 实现低风险审批的自动化，高影响任务则强制要求人工监督；
- 限制人工智能生成的通知数量，防范人类认知过载；
- 设定频率阈值，限制人工智能过度生成通知、请求与审批指令，防范决策疲劳；
- AI Agent 修改自身业务目标前，需经过双智能体协同验证；
- 为人工审核人员提供人工智能辅助的解释摘要，通过清晰、简洁的人工智能决策说明，帮助审核人员更快做出更合理的决策；
  - 利用机制可解释性框架，能有效提升该能力的规模化落地效率，详情参考：[https://arxiv.org/html/2404.14082v1](https://arxiv.org/html/2404.14082v1 "https://arxiv.org/html/2404.14082v1")
- 为人工审核人员实施自适应工作负载分配，动态平衡人工智能审核任务，避免单个审核人员出现决策疲劳。

# 被动响应：识别人工智能诱导的人类操纵行为

- 启用目标一致性验证，检测并阻断人工智能非预期的行为偏移；
- 统计各 AI Agent 的目标修改请求频率，若发现某一智能体反复尝试修改目标，需警惕操纵攻击；
- 部署人工智能行为模式基线，检测 AI Agent 输出是否存在系统性的操纵倾向，如频繁在关键决策节点提供带有偏向性的建议；
- 对人工智能生成的高风险操作建议，自动触发二次人工审核流程，防范单次审核中的认知盲区被利用；
- 检测人工智能是否存在利用社会工程学手段影响人类决策的行为，如刻意制造紧迫感、隐瞒关键信息或选择性呈现数据；
- 对人工智能与人类交互过程中出现的异常说服模式进行标记，若发现 AI Agent 反复以相似话术推动同一未授权操作，立即预警。

# 检测监控：强化人工智能决策的可追溯性与日志管理

- 推行加密日志记录与不可篡改的审计轨迹，防范日志被篡改；
- 对人工智能决策工作流实施实时异常检测；
- 监控并记录人类对人工智能建议的人工覆盖操作，分析审核人员行为模式，排查潜在偏见或人工智能目标错位问题；
- 对高风险工作流中的决策反转行为进行检测并标记 —— 若人工智能生成结果初期被驳回，却在可疑情况下被批准，需及时预警。

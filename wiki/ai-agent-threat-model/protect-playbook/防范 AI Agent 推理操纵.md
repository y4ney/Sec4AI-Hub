# 核心目标

防范攻击者操纵人工智能意图、利用人工智能欺骗性行为绕过安全防护，提升人工智能行为的可追溯性。

# 主动防护：减少攻击面，实施智能体行为画像构建

- 限制工具访问权限，最小化攻击面，防范用户交互被操纵；
- 部署验证机制，检测并过滤人工智能输出中被篡改的响应内容；
- 搭建监控能力，确保 AI Agent 的行为与其定义角色和预期操作一致，防范操纵尝试。

# 被动响应：防范 AI Agent 目标被操纵

- 启用目标一致性验证，检测并阻断人工智能非预期的行为偏移；
- 统计各 AI Agent 的目标修改请求频率，若发现某一智能体反复尝试修改目标，需警惕操纵攻击；
- 施加行为约束，防范人工智能形成自我强化循环，确保 AI Agent 不会超出预设业务参数自主调整目标。

# 检测监控：强化人工智能决策的可追溯性与日志管理

- 推行加密日志记录与不可篡改的审计轨迹，防范日志被篡改；
- 对人工智能决策工作流实施实时异常检测；
- 监控并记录人类对人工智能建议的人工覆盖操作，分析审核人员行为模式，排查潜在偏见或人工智能目标错位问题；
- 对高风险工作流中的决策反转行为进行检测并标记 —— 若人工智能生成结果初期被驳回，却在可疑情况下被批准，需及时预警；
- 检测并标记存在操纵企图、或非预期影响人类决策的人工智能响应内容。

# 威胁描述

意图破坏与目标操纵（Intent Breaking & Goal Manipulation）指攻击者利用 AI Agent 中数据与指令未分离的漏洞，通过提示注入、受污染的数据源或恶意工具，篡改 Agent 的规划、推理与自我评估过程。这使得攻击者能够推翻 Agent 的既定目标、操纵决策过程，并迫使 AI Agent 执行未授权操作，在具备自适应推理能力和外部交互能力的系统（如基于反应式推理框架的  Agent ）中，该威胁尤为突出。

该威胁利用 AI Agent在规划和目标设定能力上的漏洞，使攻击者能够操纵或重定向其目标与推理路径。常见方式之一是前文提到的“Agent劫持”。

# 与 OWASP LLM Top 10 的关联

该威胁与  LLM01:2025 Prompt injection 相关， 但 Agentic AI 中的目标操纵进一步扩大了提示注入的风险 —— 攻击者可注入对抗性目标，改变 AI Agent 的长期推理过程。

# 攻击场景

## 场景1：渐进式规划注入

攻击者通过注入隐蔽的子目标，逐步修改 AI Agent 的规划框架，使其决策逐步偏离原始目标，同时表面上仍保持逻辑推理的合理性。

## 场景 2：直接式规划注入

攻击者向聊天机器人发出指令，使其无视原始规则，转而串联工具执行操作，实施数据窃取、发送未授权邮件等未授权行为。

## 场景 3：间接式规划注入

恶意构造的工具输出中包含隐藏指令，AI 将其误判为业务目标的一部分，进而导致敏感数据泄露。

## 场景 4：反射循环陷阱

攻击者触发AI的无限或深度自我分析循环，消耗系统资源并使其无法做出实时决策，实现对系统的有效瘫痪。

## 场景 5：元学习漏洞注入

攻击者操纵AI的自我优化机制，植入特定的学习模式，逐步破坏其决策的完整性，最终使其能够执行未授权操作。

# 缓解方法

- 实施规划验证框架
- 对反思过程设置边界管理机制
- 部署动态目标对齐保护机制
- 可以通过“模型审计模型”的方式进行行为审计，即：使用另一个模型对当前Agent进行检查，识别显著的目标偏离行为

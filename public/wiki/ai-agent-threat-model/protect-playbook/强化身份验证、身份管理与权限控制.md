# 核心目标

防范人工智能未授权提升权限、身份仿冒，以及访问控制违规行为。

# 主动防护：搭建安全的人工智能身份验证机制

- 要求 AI Agent 完成加密身份验证；
- 实施精细化的基于角色的访问控制（RBAC）与基于属性的访问控制（ABAC），确保人工智能仅拥有履行其角色所需的最小权限；
- 对高权限人工智能账户部署多因素身份验证（MFA）；
- 对长时间运行的人工智能会话，强制实施持续重认证；
- 除非通过预设工作流获得明确授权，否则禁止跨智能体权限委托；
- 对人工智能间交互实施双向认证，通过双向验证防范未授权的智能体间通信；
- 限制人工智能凭证的持久化时长，确保人工智能生成的凭证为临时凭证，且短时间内过期，降低被利用风险。

# 被动响应：限制权限提升与身份继承行为

- 启用动态访问控制，使提升的权限自动过期；
- 利用人工智能驱动的行为画像，检测智能体角色分配与访问模式中的不一致性；
- 涉及身份验证修改的高风险人工智能操作，需经过双智能体协同验证或人工验证；
- 实时检测并标记角色继承异常，若发现 AI Agent 被动态授予超出其常规业务范围的角色，立即预警；
- 对权限提升施加时间限制，确保拥有高权限的 AI Agent，仅能在预批准时长内保留权限，超时后自动降级。

## 检测监控：检测并阻断人工智能仿冒尝试

1. 长期跟踪 AI Agent 的行为，检测身份验证中的不一致性；
2. 监控 AI Agent 的非预期角色变更或权限滥用行为；
3. 对 AI Agent 在正常业务范围外发起高权限操作的异常行为进行标记；
4. 将人工智能身份验证结果与历史访问趋势关联分析，对比认证尝试与过往访问日志，检测可疑偏差；
5. 实施身份偏移监控，若发现 AI Agent 的行为与其历史活动不符，立即标记；
6. 监控并标记反复的身份验证失败尝试，识别 AI Agent 或用户的多次未授权登录行为 —— 此类行为可能预示凭证暴力破解攻击；
7. 检测并标记跨智能体触发的串联或递归工具执行模式，此类行为可能表明攻击者通过协议滥用实现权限提升；对生成可疑协议流量的智能体实施隔离（如通过 MCP 发起的工具调用突发激增）。
